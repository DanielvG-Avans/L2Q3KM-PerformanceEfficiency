\section{Introduction}

Modern front-end applications commonly use either client-side rendering (CSR) or server-side rendering (SSR). In practice, this choice is often based on metrics such as loading speed, interactivity, and search engine optimization (SEO). However, for mobile usage scenarios, resource efficiency is also critical because prolonged CPU and memory pressure can negatively impact battery life and device responsiveness.

Existing comparisons between SSR and CSR focus primarily on performance as speed. Fewer studies evaluate performance as resource efficiency, and even fewer do so in a controlled mobile context. As a result, it remains unclear which rendering strategy imposes less computational load on the client device during rendering and interaction.

This report addresses that gap by comparing SSR and CSR within one functionally comparable Next.js prototype. The analysis centers on client-side energy use in a mobile benchmark, complemented by selected browser-facing metrics and client-side resource indicators such as JavaScript heap sampling. The objective is to provide evidence-based guidance for selecting a rendering strategy when mobile resource efficiency matters alongside user-facing performance.
