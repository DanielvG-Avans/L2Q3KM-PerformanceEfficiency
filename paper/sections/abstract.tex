\begin{abstract}
This study investigates the influence of server-side rendering (SSR) and client-side rendering (CSR) on resource efficiency in a mobile context. Although these render strategies are frequently compared using speed and user-experience metrics, limited attention is given to CPU load, memory usage, and their relation to energy consumption.

To address this gap, one web application was implemented in two functionally identical variants: SSR and CSR. Under controlled test conditions, CPU usage and memory usage were measured during initial rendering and interaction moments. The collected data was analyzed using descriptive statistics to identify differences between both strategies and to evaluate whether CPU metrics can be used as a practical proxy for energy consumption.

The report contributes to more informed render-strategy decisions in mobile front-end development by shifting the evaluation focus from performance as speed to performance as resource efficiency.

\keywords{server-side rendering, client-side rendering, mobile energy efficiency, CPU usage, memory usage, web performance}
\end{abstract}
